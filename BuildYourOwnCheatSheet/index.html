HTML Doc Setup

This will be an outline of the basic document setup for starting an HTML Document. This will cover the DOCTYPE, html, head, body, header, main and footer tags that are apart of css and make the general skeleton of a html document.

Element names are surrounded by angled brackets.

Semantics

  • A key goal to focus on when creating your html document is to be as semantic as possible. That means we want to use the correct tags that will accurately describe the content that is being used inside the tags. An example would be using a header tag to wrap around a navbar tag. This will help our user understand the outline of the document. Most users will not be using the "inspect" feature on browsers, but the ones that do will be able to get a sense of what the document is supposed to look like.
  • Semantics are key because they help search engines use keywords in order to find your website.
  • They help screenreaders navigate a page for visually impaired users.
  • It is easier for a developer that is going through your code to see an article and understand why an article tag is there, instead of a div tag that gives no semantic meaning besides it being a block-level element.

HTML Doc Tags Explained

  • <DOCTYPE>: A required tag. Requried tag in order for the document to behave correctly.
  • <html>: Known as the root element. Wraps all the content on the entire page.
  • <head>: Acts as the container for all the content that will not be displayed visually on your html document. This includes things like keywords, links to stylesheets, page descriptions for search results.
  • <body>: This contains all the content that you want the user to see when they visit your page. Most of your html writing will be inside a body tag.
  • <header>: Represents introductory content, typically consists of a logo and a navigation bar.
  • <main>: The dominant content of the <body> tag. Consists of content that is directly related to or expands upon the central topic of a document.
  • <footer>: Contains information about the author of the content, links to related documents or copyright information.

HTML Tables

Tables can make organization on an HTML doc a breeze once mastered. Remembering the general structure of a table can make creating tables from scratch a breeze.

General Structure

  • The general structure of a table is to initialize the table with the <table> tag. This will wrap around your future table elements.
  • The MDN doc recommends the rest of the order:
    1. An optional <caption>.
    2. zero or more <colgroup> elements.
    3. an optional <thead> element.
    4. either one of the following:
      • zero or more <tbody> elements.
      • one or more <tr> elements.
    5. an optional <tfoot> element.

Table Tags Explained

  • <table> signify the creation of a table in HTML. This will wrap around all the elements in the user's table.
  • <caption>: Considered optional, the value of <caption> will seek to describe the table's purpose, and help the user decide if they need to read the rest of the content in the table or skip over it.
  • <colgroup>: Useful for applying styles to entire columns, instead of repeating the styles for each cell, for each row. Must be a child of a table element, after any caption elements, and before any thead, tbody, tfoot, and tr elements. To define different properties to a column within a colgroup, use the col tag within the colgroup tag.
  • <thead>: Creates a row that will illustrate what will become of the columns.
  • <tbody>: Encapsulates a set of table rows(<tr> elements), indicating to the user that this is the body of the table.
  • <tr>: Defines a row of cells in a table. These cells can be established using a mix of <td> (data cells) and <th> (header cell) elements.
  • <tfoot>: Defines a set of rows summarizing the columns of the table.

CSS Selectors & Specificity

Let's understand more about CSS Selectors & Specificity. Let us take a look at classes and ids and understand when to use each one.

In order for our HTML documents to have cool colors and cool backgrounds, we have to add CSS to our document. Cascading Style Sheets, abbreviated CSS, gives our doc the oomph that makes the user enjoy the page they are looking at. If proper HTML gives the user a clear understanding of what the document is, then proper CSS styles the page so the user does not get bored AND can properly read the webpage. Imagine, green font with a white background. Eww.

To start with CSS, there are basic properties that will be explained later on. First, we need to understand how to apply the properties to the page. The first important topic that needs to be discussed is CSS selectors & specificity. Let's say we want to make the headline of our webpage a certain color. We need to understand how to select just that headline, that h1 tag, and make it the color blue without anything else changing to the color blue. That is where css selectors and specificity comes into play.

Types of Selectors

  • Universal Selector: Selects all elements.
    • Syntax: * {}
  • Type Selectors: Selects all elements that have the given tag name.
    • Syntax: elementname
    • Example: h1 will match any h1 element in the document.
  • Class selectors: Selects all classes with the given name. Must have a . before the name of the selector.
    • Syntax: .classname
    • Example: .headline will select all elements that have the class headline.
  • ID selectors: Selects the element with the ID name. Only one element can have that specific ID name. This is known to be the most specific way to target an element when using CSS.
    • Syntax: #idname
    • Example: #hello will match the element with an id name of hello.
  • Attribute Selectors: Selects all elements with the given attribute.
    • Syntax: [attr]
    • Example: [autoplay] will match all elements that have the autoplay attribute set (to any value).

Combinators

Combinators are ways for the developer to target a certain element that is nested inside an element.

  • Descendant combinator: The space will select nodes that are descendents of the first element.
    • Syntax: A B {}
    • Example:A B will match all <A> elements that are nested inside a <B> element.
  • Child combinator: The > will select nodes that are direct children of the first element.
    • Syntax: A > B {}
    • Example: ul > li will match all <li> elements inside a <ul> element.
  • General sibling combinator: The ~ combinator selects siblings. This means that the second element will follow the first and that they both share the same parent selector.
    • Syntax: A ~ B {}
    • Example: p ~ span will match all <span> elements that follow a <p> element, immediately or as long as they share the same parent element.
  • Adjacent sibling combinator: The + will select adjacent siblings. This means that the second element directly follows the first, and both share the same parent.
    • Syntax: A + B {}
    • Example: h2 + p will match all <p> elements that DIRECTLY follow an <h2> element.
  • Column combinator: The || combinator will select nodes which belong to a column. Used in tables.
    • Syntax: A || B {}
    • col || td will match all <td> elements that belong to the scope of <col>

Note: Understanding CSS selectors and the use of combinators will allow you to be consise and precise with selecting which element you want to specifically target for CSS.

Understanding Specificity

Specificity is based on the matching rules that are composed of different types of CSS selectors. As stated earlier, ID is the most specific you can get. If you use an ID tag on an element, and then style it, it will overide any other CSS selector, such as a class selector on the same element. Similarily, class selectors will overide any type selector. There is an exception, !important, which when placed on a property, will override any other declaration.
