HTML Doc Setup

This will be an outline of the basic document setup for starting an HTML Document. This will cover the DOCTYPE, html, head, body, header, main and footer tags that are apart of css and make the general skeleton of a html document.

Element names are surrounded by angled brackets.

Semantics

  • A key goal to focus on when creating your html document is to be as semantic as possible. That means we want to use the correct tags that will accurately describe the content that is being used inside the tags. An example would be using a header tag to wrap around a navbar tag. This will help our user understand the outline of the document. Most users will not be using the "inspect" feature on browsers, but the ones that do will be able to get a sense of what the document is supposed to look like.
  • Semantics are key because they help search engines use keywords in order to find your website.
  • They help screenreaders navigate a page for visually impaired users.
  • It is easier for a developer that is going through your code to see an article and understand why an article tag is there, instead of a div tag that gives no semantic meaning besides it being a block-level element.

HTML Doc Tags Explained

  • <DOCTYPE>: A required tag. Requried tag in order for the document to behave correctly.
  • <html>: Known as the root element. Wraps all the content on the entire page.
  • <head>: Acts as the container for all the content that will not be displayed visually on your html document. This includes things like keywords, links to stylesheets, page descriptions for search results.
  • <body>: This contains all the content that you want the user to see when they visit your page. Most of your html writing will be inside a body tag.
  • <header>: Represents introductory content, typically consists of a logo and a navigation bar.
  • <main>: The dominant content of the <body> tag. Consists of content that is directly related to or expands upon the central topic of a document.
  • <footer>: Contains information about the author of the content, links to related documents or copyright information.

HTML Tables

Tables can make organization on an HTML doc a breeze once mastered. Remembering the general structure of a table can make creating tables from scratch a breeze.

General Structure

  • The general structure of a table is to initialize the table with the <table> tag. This will wrap around your future table elements.
  • The MDN doc recommends the rest of the order:
    1. An optional <caption>.
    2. zero or more <colgroup> elements.
    3. an optional <thead> element.
    4. either one of the following:
      • zero or more <tbody> elements.
      • one or more <tr> elements.
    5. an optional <tfoot> element.

Table Tags Explained

  • <table> signify the creation of a table in HTML. This will wrap around all the elements in the user's table.
  • <caption>: Considered optional, the value of <caption> will seek to describe the table's purpose, and help the user decide if they need to read the rest of the content in the table or skip over it.
  • <colgroup>: Useful for applying styles to entire columns, instead of repeating the styles for each cell, for each row. Must be a child of a table element, after any caption elements, and before any thead, tbody, tfoot, and tr elements. To define different properties to a column within a colgroup, use the col tag within the colgroup tag.
  • <thead>: Creates a row that will illustrate what will become of the columns.
  • <tbody>: Encapsulates a set of table rows(<tr> elements), indicating to the user that this is the body of the table.
  • <tr>: Defines a row of cells in a table. These cells can be established using a mix of <td> (data cells) and <th> (header cell) elements.
  • <tfoot>: Defines a set of rows summarizing the columns of the table.
